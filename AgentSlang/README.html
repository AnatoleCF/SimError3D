AgentSlang

Installation

Along with this document, the archive that you have downloaded contains a few folders. You mainly need to concern yourself with these:

bin/
This contains the startup script for your platform ( AgentSlang.bat for Microsoft Windows, AgentSlang for Unix-based OSes).
Run it without any option to see the full list of available parameters.
config/
This directory contains several configuration files that each describe a set of components and their connections.
javadoc/
This contains documentation files for various AgentSlang components.
Nareca/
This contains story data.

Available Components

Many components are available for use, although some of them might require specific setup procedures to be carried out. Such procedures will typically be described in the documentation for the component.

Non-exhaustive component list

GoogleASRComponent
A component that enables the use of vocal recognition.
VCStreamer / VCStreamer
Components for videoconferencing.
FaceReaderComponent
Component for using FaceReader data from other AgentSlang components.
StoryGraphComponent
Can be used in conjunction with storytelling components to control them using a graphic display of the story.

Running the platform

Once you have decided which configuration file to run, you can simply pass it to the startup script along with the appropriate profile.

For example, if you use Linux and want to use config-nareca.xml with the (only) profile profile1, you will want to run the following command from the bin directory: 

./AgentSlang -config ../config/config-nareca.xml -profile profile1

Note that in the case of multi-machine configurations, you may have to adjust the IP addresses in the corresponding cnsService-*.xml file.

Included configuration files

config/config-nareca.xml
Simple config file that you can use to test simple story narration.
config/config-nareca-one-machine.xml
More elaborate config file that narrates the story using MARC, can be controlled via a graph display and can optionally use vocal recognition (see the documentation on how to set it up).
config/config-nareca-two-machines.xml
Similar to the previous configuration, but runs on two machines: it is a Wizard of Oz experiment in that one machine is the subject and the other one is the operator.
Their respective profiles in the configuration file are profile-subject and profile-operator.
Note: MARC must be started first (using the Narrative.xml settings file), then profile-subject, and finally profile-operator.
config/config-nareca-four-machines.xml
A prototype for a configuration that would involve four participants (two avatars, and two agents).
It uses the following profile names:
  • profile-subject-1: must be started first, but after two instances of MARC (running Narrative.xml and Narrative2.xml)
  • profile-subject-2: must be started after the first subject and two instances of MARC (running the same settings as for profile-subject-1)
  • profile-operator-1 and profile-operator-2: must be started after the first subject and after one instance of MARC (running Narrative.xml)
